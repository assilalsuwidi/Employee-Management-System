# 4. واجهات المستخدم، أمان الفرونت إند، وطريقة التشغيل الكاملة

في هذا القسم الأخير، سنقوم بشرح واجهات المستخدم (Frontend)، كيف تم تصميمها وحمايتها، وكيفية تشغيل المشروع خطوة بخطوة بالكامل.

---

## 💻 واجهات المستخدم (UI Design)

تصميم النظام يعتمد على مبدأ الـ **Glassmorphism** (التصميم الزجاجي المضبب) فوق بيئة داكنة (Dark Mode)، مما يعطي شكلاً عصرياً جداً ويقلل من إجهاد العين.

### أهم الواجهات:
1. **واجهة تسجيل الدخول (Login Interface)**:
   - واجهة نظيفة وبسيطة تحتوي على حقلي اسم المستخدم وكلمة المرور.
   - تقوم بالتحقق من المدخلات (Client-Side Validation) قبل إرسالها للخادم.
2. **لوحة التحكم (Dashboard)**:
   - شريط تنقل جانبي (Sidebar) وشريط علوي.
   - تعرض إحصائيات سريعة للرؤساء عن الموظفين والمهام.
3. **واجهات الإدارة (الموظفين، المجموعات، الحضور، الرواتب)**:
   - تعتمد على عرض البيانات في جداول (Tables) مبنية بطريقة ديناميكية لتسهيل البحث.
   - تحتوي على نوافذ منبثقة (Modals) عائمة لإضافة عناصر جديدة، لكي لا يضطر المستخدم لمغادرة الصفحة الحالية.

## 🛡 الأمان في الواجهة الأمامية (Frontend Security)

حماية الواجهة الأمامية لا تقل أهمية عن حماية الواجهة الخلفية، وقد شملت الإجراءات التالية:

1. **إمكانية الوصول والتباين (Accessibility & Contrast)**:
   - تم فحص وتصحيح التباين اللوني (Contrast Ratio) باستخدام معايير WCAG. تم التأكد من أن النصوص البيضاء واضحة جداً فوق الخلفيات الملونة والشفافة لضمان راحة ومقروئية عالية لجميع المستخدمين.
2. **إغلاق ثغرات النماذج (Label & Form Security)**:
   - تم التأكد من ربط جميع تسميات النماذج (Labels) بحقول الإدخال (Inputs) الخاصة بها باستخدام خاصية `htmlFor`. هذا الإجراء يمنع المشاكل الأمنية للنماذج ويضمن قراءة صحيحة بواسطة برامج قارئ الشاشة (Screen Readers).
3. **تخزين الـ Token بشكل آمن**:
   - يتم التعامل مع توكن المصادقة بحذر وإرساله بشكل نظيف في ترويسات طلبات `fetch` فقط لضمان عدم تعرضه للسرقة عبر الثغرات البرمجية.
4. **حماية التوجيه (Protected Routes)**:
   - لا يمكن للمستخدم العادي أو الزائر رؤية لوحة التحكم. الـ Frontend يتحقق من وجود الـ Token في المتصفح، وإذا كان غير موجود أو منتهي الصلاحية، يقوم بطرد المستخدم آلياً إلى صفحة الدخول (Login).
5. **منع هجمات XSS**:
   - بيئة React بطبيعتها تقوم بتنظيف كافة البيانات القادمة من الواجهة الخلفية (Sanitization) قبل عرضها على الشاشة، مما يمنع تنفيذ أي كود جافاسكربت خبيث قد يتم إدخاله في قاعدة البيانات (XSS Attack).

## 🚀 كيفية تشغيل المشروع بالكامل (Execution Guide)

الخيار الأفضل والموصى به لتشغيل هذا المشروع هو استخدام **Docker**، لأنه يتكفل بإعداد قواعد البيانات والخوادم من تلقاء نفسه دون أي تداخل.

### الخطوة 1: تجهيز الملفات البيئية
1. اذهب إلى مجلد الواجهة الخلفية `backend/`.
2. ستجد ملفاً اسمه `.env.example`. قم بنسخه وإعادة تسمية النسخة إلى `.env`.
3. هذا الملف يحتوي على إعدادات قاعدة البيانات والمفتاح السري لـ JWT.

### الخطوة 2: التشغيل باستخدام Docker
1. قم بفتح سطر الأوامر (Terminal) في المجلد الرئيسي للمشروع (المجلد الذي يحتوي على ملف `docker-compose.yml`).
2. اكتب الأمر التالي:
   ```bash
   docker-compose up -d --build
   ```
3. انتظر قليلاً، سيقوم دوكر بإنشاء 3 حاويات:
   - حاوية لقاعدة البيانات (MySQL).
   - حاوية للواجهة الخلفية (Flask API).
   - حاوية للواجهة الأمامية (React).

### الخطوة 3: تصفح النظام
بمجرد الانتهاء من البناء:
- **الواجهة الأمامية للمستخدم** ستكون متاحة على الرابط: `http://localhost`
- **واجهة برمجة التطبيقات (API)** ستكون متاحة على الرابط: `http://localhost:5000/api`

### 💡 التشغيل المحلي (للمطورين بدون دوكر)
إذا لم يكن لديك دوكر، يمكنك التشغيل يدوياً:
1. قم بتشغيل سيرفر محلي لـ MySQL وقم بإنشاء قاعدة بيانات.
2. ادخل إلى مجلد `backend` وشغل الأوامر التالية:
   ```bash
   pip install -r requirements.txt
   python run.py
   ```
3. ادخل إلى مجلد `frontend` وشغل الأوامر:
   ```bash
   npm install
   npm run dev
   ```
   (ستكون الواجهة الأمامية متاحة على منفذ 5173).
